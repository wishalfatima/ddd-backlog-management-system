
1. USE CASE DIAGRAM


@startuml
left to right direction
actor ProductOwner
actor Developer
actor ScrumMaster

rectangle BacklogSystem {
    ProductOwner --> (Add Backlog Item)
    ProductOwner --> (Update Backlog Item)
    ProductOwner --> (Assign Backlog Item to Release)
    ProductOwner --> (Assign Backlog Item to Product)
    ScrumMaster --> (Assign Backlog Item to Sprint)
    Developer --> (View Backlog Item)
    Developer --> (View Related Release Items)
}
@enduml


2. UML COMPONENT DIAGRAM


@startuml

  [Backlog Component] --> [Product Management Service]
  [Backlog Component] --> [Release Management Service]
  [Backlog Component] --> [Sprint Management Service]
}

[Product Management Service] --> [Product Context]
[Release Management Service] --> [Release Context]
[Sprint Management Service] --> [Sprint Context]

note right of [Backlog Component]
  Handles BacklogItem creation, update,
  and assignment to Product, Release, Sprint.
end note
@enduml

@startuml

class BacklogSystem {
    +addBacklogItem(item: BacklogItem): void
    +updateBacklogItem(itemId: string, updatedItem: BacklogItem): void
    +assignToProduct(itemId: string, productId: string): void
    +assignToRelease(itemId: string, releaseId: string): void
    +assignToSprint(itemId: string, sprintId: string): void
    +viewBacklogItem(itemId: string): BacklogItem
    +getRelatedReleaseItems(itemId: string): List<ReleaseItem>
}

class BacklogItem {
    +id: string
    +summary: string
    +story: string
    +storyPoints: int
    +type: string
    +status: string
}

    +id: string
  
}

    +id: string
}

    +id: string
}
